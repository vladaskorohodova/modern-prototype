# Button Overview

The Button component is a fundamental interactive element used throughout your application. It supports multiple variants and states to fit various design needs.

## Basic usage

The Button component comes in three variants: solid, outline, and ghost. Each variant conveys a different level of emphasis.

## Accessibility

The Button component is built with accessibility in mind:

- Proper semantic HTML using `<button>` element
- Full keyboard support (Space and Enter keys)
- Clear focus indicators
- Disabled state properly communicated to screen readers
- Sufficient color contrast in all variants

## Usage guidelines

### When to use

- Use **solid** buttons for primary actions (e.g., "Submit", "Save")
- Use **outline** buttons for secondary actions (e.g., "Cancel", "Back")
- Use **ghost** buttons for tertiary actions or when you need minimal visual weight

### Best practices

- Keep button text short and action-oriented
- Ensure buttons have sufficient padding for touch targets (minimum 44x44 pixels)
- Don't use too many primary buttons on a single page
- Provide visual feedback on hover and active states
